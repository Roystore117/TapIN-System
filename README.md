# Tap-IN

**美容院のための、タップひとつで完結する勤怠・人事ダッシュボード。**

現場のスタッフは「名前を選んでタップ」するだけで出退勤。
オーナーはブラウザひとつで、**勤怠・公休・給与設計・キャリア支援・月次集計** まで一画面で管理できます。
データの本体は Notion に直接書き込まれるので、「アプリで入れた情報を、Notion でそのまま分析・編集できる」二刀流の運用が可能。
専用サーバーも複雑な管理画面も不要。**美容師の手元と、オーナーの Notion を最短距離でつなぐ** のが Tap-IN です。

> 最終更新：2026年4月18日

---

## 🎯 主要機能

### 打刻画面（スタッフ用）
- 名前を選んでワンタップで出退勤
- 誤打刻防止の確認ステップ
- 出退勤デフォルトの自動判定（今日の状態から推測）
- 二重出勤・不正退勤の自動ブロック
- 保健師の一言をランダム表示（福利厚生 UX）

### 管理者画面（オーナー用・7タブ構成）

| タブ | 機能 |
|---|---|
| 📅 **月次集計** | 従業員別の勤怠を月単位で一覧・編集・CSV出力（開発中） |
| 🏖 **公休設定** | カレンダーUIで公休・有給をドラッグ&ドロップで設定 |
| 🌱 **キャリア支援** | ランク進捗バー・タイムライン・正社員転換条件を可視化 |
| 💴 **店舗別設定** | 始業／終業時刻・みなし残業・アラート閾値など店舗別に設定 |
| 👥 **従業員マスタ** | 18項目の従業員情報を CRUD 管理（基本情報 / キャリア情報タブ） |
| 💬 **保健師の一言** | 打刻画面に表示されるメッセージの管理 |
| 🛠 **デバッグ** | 開発用モック時刻設定（本番前に削除予定） |

## 🧭 各機能の詳細

### 打刻画面（`/`）

**フロー：**
1. 従業員選択 → 今日の状態から出退勤デフォルトが自動セット
2. ボタンタップ → 確認画面（誤タップ防止）
3. 「打刻する」で Notion に書き込み

**ヒューマンエラー対策：**

| ケース | 挙動 |
|---|---|
| 二重出勤 | ブロック +「出勤中です」と表示 |
| 退勤忘れ後の退勤 | ブロック + オーナーへの連絡を促す |
| 本日の出勤記録なしの退勤 | 同上 |

出勤忘れは意図的に Tap-IN では補完せず、**オーナーが Notion で直接入力する運用**。

---

### 月次集計タブ

- 従業員・月を選択すると、その月の勤怠レコードが一覧表示
- セルをクリックで直接編集（日付・出勤・退勤・休憩・勤務状態・備考）
- 実働時間は Notion の数式で自動計算
- **CSV出力は開発中**（モーダルで案内）

---

### 公休設定タブ

**3パネル構成：**
- 左：定休曜日セレクター + 従業員リスト（公休・有給カウント表示）
- 右：カレンダー

**操作：**
- 従業員の名前ピルをカレンダーにドラッグ&ドロップで公休登録
- カレンダー上のピルをクリックで削除、ドラッグで別日へ移動
- 公休 / 有給 の種別選択はポップオーバーで
- 定休曜日は店舗設定DB に保存され、全カレンダーに即反映

---

### キャリア支援タブ

**2つのサブタブ：**
1. **キャリアアップ対象** — 対象者の個別ダッシュボード（名前リスト + 詳細）
2. **全従業員一覧** — 全員の表形式一覧、行クリックで詳細編集

**ダッシュボード構成：**
- **PROFILE** — 名前・部門・指名料・キャリアアップ対象・新卒/中途
- **RANK** — Lv1〜Lv7 の進捗バー（現在位置が視覚的にわかる）
- **TIMELINE** — 入社 / キャリアアップ面談 / 正社員切替 を時間軸で可視化（NOWマーカー付き）
- **CONDITIONS** — 試用期間・転換予定日・試用期間給与・正社員給与・賃金上昇率
- **編集モード** — 編集ボタンで全項目を編集可（未保存ガード機能付き）

**アラート表示：**
- 30日以内の面談・切替 → 青リング表示
- 7日以内 → 青塗りドット（緊急）
- 左リストは**直近のイベントが近い順**にソート

---

### 店舗別設定タブ

- 始業・終業時刻の標準値
- みなし残業時間
- 時間外アラート閾値
- 店舗ごとに個別設定可能（店舗設定DB に保存）

---

### 従業員マスタタブ

**2つのサブタブ：**
1. **基本情報** — 従業員識別子・番号・姓・名・事業所・部門・職種・契約・状態
2. **キャリア情報** — ランク・指名料・キャリアアップ対象・入社日・更新時期 など

**機能：**
- 横スクロール対応の表形式（sticky ヘッダー）
- 行クリックでモーダル編集
- 新規追加・既存編集対応
- 部門名は店舗設定DBへの Relation で管理（名前ミスマッチを物理的に防止）

---

### 保健師の一言タブ

- 打刻画面にランダム表示されるメッセージの管理
- 有効／無効のトグル
- 新規追加・編集

## 🗂 Notion DB 構成

Tap-IN は **5つの Notion DB** を連携して動作します。

### ① 勤怠ログDB (`TIMELOG_DB_ID`)

1人1日1レコード。出勤時に新規作成、退勤時に同レコードを更新。

| プロパティ | 型 | 用途 |
|---|---|---|
| タイトル | タイトル | `YYYY-MM-DD 名前` |
| ②従業員マスタDB_RM03P | リレーション | 従業員との紐付け |
| 日付 | 日付 | 打刻日 |
| 出勤 / 退勤 | 日時 | 打刻時刻 |
| 休憩 | セレクト | デフォルト `2.5` |
| 実働 | 数式 | 自動計算：`退勤 - 出勤 - 休憩` |
| 勤務状態 | セレクト | 出勤 / 公休 / 有給 |
| 備考 / 承認 | テキスト / 選択 | 任意 |

### ② 従業員マスタDB (`EMPLOYEE_DB_ID`)

18項目。基本情報 + キャリア情報。

**基本情報：** 従業員識別子（title）・従業員番号・姓・名・事業所名・部門名（Relation → 店舗設定DB）・職種名・契約職種・ステータス

**キャリア情報：** ランク・指名料・キャリアアップ対象・中途or新卒・入社時期・キャリアアップ更新時期・キャリアアップ面談時期・試用期間給与・正社員給与

### ③ 店舗設定DB (`STORE_SETTINGS_DB_ID`)

店舗マスタ兼設定値。

| プロパティ | 型 | 用途 |
|---|---|---|
| 店舗名 | タイトル | 店舗識別 |
| 定休曜日 | セレクト | 公休カレンダーに反映 |
| 始業標準時刻 / 終業標準時刻 | テキスト | 給与計算基準 |
| みなし残業時間 / アラート閾値 | 数値 | 時間外管理 |

### ④ 保健師の一言DB (`TIPS_DB_ID`)

打刻画面で表示するランダムメッセージ。

| プロパティ | 型 |
|---|---|
| 一言 | タイトル |
| 有効 | チェックボックス |

### ⑤ 時間外申請DB (`OVERTIME_REQUEST_DB_ID`)

時間外申請の記録（打刻画面からの送信先）。

## 📐 運用ルール

- **出勤打刻し忘れ** → Notionで直接入力。Tap-IN では補完しない
- **時刻修正** → 月次集計タブのインライン編集 or Notionで直接編集
- **公休登録** → 公休設定タブでドラッグ&ドロップ（または Notion で直接 `勤務状態=公休`）
- **店舗の追加** → 店舗設定DB にページ追加 → 全画面に自動反映（コード変更不要）
- **ランク・契約種別などの選択肢変更** → Notion の select オプションを編集 → 全画面に自動反映（コード変更不要）

## ⚠️ 既知の制限事項（2026年4月時点）

本番運用前に対応予定：

- **認証**：単一管理者パスワード（cookie に生値保存） → セッションID方式に要改修
- **権限管理**：現状は全機能フルアクセスの単一ロール → ロール分け（オーナー/店長/経理）は未実装
- **監査ログ**：誰がいつ何を変更したかの追跡機能は未実装
- **打刻競合対策**：短時間連打で重複レコード作成の可能性（稀）
- **CSV出力**：開発中（モーダルで案内）
- **サーバーサイド入力検証**：UI依存。API直叩きはバリデーションなし

これらは**デモ・パイロット運用では許容範囲**として意図的に簡略化されています。本格運用の段階で順次対応。

## 🧠 設計哲学

- **Notion を信頼**：データの正本は Notion。Tap-IN はそれを見やすく・操作しやすくするレイヤー
- **シングルソースオブトゥルース**：選択肢・店舗名は Notion DB から動的取得、ハードコードしない
- **美容師ファースト**：打刻画面は一切の迷いなくタップ1回で完了する設計
- **オーナーの意思決定を助ける**：キャリア支援・アラートで「やるべきこと」が一目でわかる
- **ミニマル＆洗練**：白・黒・青のみで構築。情報密度と可読性のバランスを重視
